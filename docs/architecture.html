1. 概要と設計原則

ClawnPet は、デスクトップ最前面に常駐するカニのマスコット「Clawn くん」が、Claude Desktop / Claude Code のセッション活動（プロンプト送信・ツール実行・応答到着）をリアルタイムに表情・吹き出し・音声で実況するアプリです。

📖 読み取りのみ

Claude アプリが元々ローカルに書き出すファイルを tail するだけ。ネットワーク送信・プロセス介入・UI 自動操作は一切しない（VOICEVOX への localhost リクエストを除く）。

🧩 依存ゼロ

Swift + AppKit + AVFoundation のみ。swiftc 一発でビルドでき、パッケージマネージャも Xcode プロジェクトも不要。

⏱ 1 秒ポーリング

FSEvents ではなく単純なポーリング + 差分 tail。堅牢でデバッグしやすく、消費リソースも無視できる規模（stat 数百回/秒以下）。

🎭 セッション = Brain

セッションごとに独立した状態マシン（PetBrain）を持ち、メインの Clawn は「直近アクティブなセッション」を実況する。

2. 全体アーキテクチャ

すべてのウォッチャーはバックグラウンドの serial queue でポーリングし、イベントは DispatchQueue.main に載せ替えてから Router に渡します。UI と状態マシンはメインスレッドのみで触るため、ロックは不要です。

TailReader（差分 tail の要）

1. 前回オフセットから追記分だけ read（最大 2MB/回）
2. 改行で完全な行に分割し、末尾の未完行は leftover として持ち越す
3. ファイルサイズ < オフセット を検知したらローテーションとみなし 0 から読み直す
4. 新規追跡時は tailBytes だけ末尾から遡り、20 秒より古い行はパース側で捨てる

6. イベントルーティング

Router（AppDelegate）は sessionId（= transcript のファイル名）ごとに SessionTrack を生成し、イベントを対応する PetBrain に配送します。

  • primary 選定: lastEvent が最新の Track がメインの Clawn を駆動する。セッションが 1 つもない時は defaultBrain（挨拶用）を表示。
  • pending prompt 昇格: history.jsonl のプロンプトは transcript より先に届くことがある。未知の sessionId なら 15 秒だけ保留し、transcript 側の userMessage が来た瞬間に本文付きの userPrompt へ昇格させる。これで新規セッションの 1 通目から本文を吹き出しに出せる。
  • prune: 30 分イベントがない Track は破棄（カードも消える）。追跡は直近更新の transcript 最大 6 本。
  • Desktop イベント: main.log の sessionId（local_*）は transcript の sessionId と対応しないため、primary Track に配送する。
  • サブエージェント: isSidechain: true の行はツール実行のみ拾い、「(サブエージェント)」の接頭辞を付ける。
  • セッションへジャンプ: カードクリックで claude://resume?session=<uuid> を開く。Claude Desktop が transcript からセッションを取り込み、該当セッションにフォーカスする（実測: Imported CLI session … / setFocusedSession）。パラメータ名が session であることは app.asar のディープリンクハンドラから特定した。

8. 音声実況（SpeechManager）

  • エンジン: VOICEVOX（http://127.0.0.1:50021, 既定話者 ID 3 = ずんだもん）→ 不在時は自動でシステム音声（AVSpeechSynthesizer, ja-JP）へフォールバック。メニューから OFF / システム / VOICEVOX を切替可能。
  • VOICEVOX API フロー: POST /audio_query?speaker=3&text=… → 返ってきた JSON をそのまま POST /synthesis?speaker=3 へ → WAV を AVAudioPlayer で再生。
  • レート制限: 通常発話は 7 秒間隔。応答到着（celebrating）だけは割り込みフラグ付きで必ず喋る。
  • マルチセッション時: 2 セッション以上追跡中はプロジェクト名を接頭辞に付ける（「pet、へんじがきたよ！」）。
  • セッションごとの声: 「セッションごとに声を変える」ON でプロジェクト名のハッシュから VOICEVOX のインストール済み話者を安定的に割り当て（同じプロジェクトは常に同じ声）。
  • 死活監視: 30 秒ごとに GET /version（タイムアウト 1.2 秒）で VOICEVOX の生存確認。
  • 通知センター: 応答到着は Notifier 経由で macOS 通知にも出せる（4 秒レート制限、通知タップで claude://resume ジャンプ）。

10. 制約と拡張ポイント

  • claude.ai Web チャット未対応: Desktop 内のブラウザチャットはローカルにメッセージを書き出さない。残る道は CDP（--remote-debugging-port）で、実行中はシングルトンロックのため検証不可だった。検証手順と設計案は FEASIBILITY.md の CDP 章を参照。
  • assistant 本文はブロック確定単位: トークン単位のストリーミング途中経過は transcript に書かれない。
  • スケジュールタスク名は取れない: main.log にはライフサイクル（Initialized 等）しか出ず、タスク名はサーバー側。
  • 拡張案: CDP ウォッチャー（opt-in）/ カードクリックでプロジェクトを開く / 応答完了の通知センター連携 / セッションごとの担当ボイス切替。
