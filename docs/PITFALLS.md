# PITFALLS — 開発でハマった点と対策

ClawnPet の開発中に実際に踏んだ罠の記録です。同種のアプリ
（フローティング常駐ペット / ローカルファイル監視 / ad-hoc 配布）を作る人と、
将来このコードを触るエージェントのための備忘録。

## 1. 再起動のたびにペットが右へ 144px ずれる

**症状**: デプロイ（再起動）するたびにカニの位置が少しずつ右へ移動する。

**原因**: ウィンドウ位置を「左下 origin」で保存していた。起動時はウィンドウを
展開幅 260px で作ってから閉サイズ 116px に縮めるため、
「閉状態で保存した origin」に「幅 260 のウィンドウ」を復元 → 右端固定で縮小、
の過程で右端が `260 - 116 = 144px` 移動していた。

**対策**: 保存・復元を**右下角**（`frame.maxX` / `frame.minY`）基準に変更し、
起動時から閉サイズでウィンドウを作る。ユーザーから見える「カニの位置」は
右下角に張り付いているので、アンカーもそこに合わせるのが正解だった。
検証は CGWindowList の bounds 実測で「開閉・再起動とも右下角不変」を確認。

## 2. 開閉するたびカニが一瞬横を向く

**症状**: カードを開閉した瞬間、移動していないのにカニが「ぐりん」と横を向く。

**原因**: 「移動方向を向く」演出がウィンドウ `origin.x` のフレーム間差分で
速度検出していた。開閉でウィンドウ幅が変わると右端固定のため origin.x が
±144px 動き、それを高速移動と誤検知していた。

**対策**: 差分の追跡対象を `origin.x` から **`maxX`（右端）** に変更。
右端はドラッグでしか動かないので、開閉の伸縮では発動しなくなる。

## 3. 開閉でマスコットの大きさが変わってしまう

**症状**: 閉 0.52 倍 / 開 0.74 倍の 2 つの縮尺で描いていたため、
開閉のたびにカニの大きさが変わり、ユーザー体験が悪かった。

**対策**: ChatGPT Desktop のペットと同じ構造に刷新。カニは開閉どちらも
**右下 116×112 領域に 0.52 倍で描く単一パス**とし、開いたときは
その上にカードスタックと ˅ ボタンが乗るだけにした。
「状態によって描画パスが分かれる」設計はサイズ・位置の不一致バグの温床になる。

## 4. 通知センターの許可が下りない（ad-hoc 署名）

**症状**: `UNUserNotificationCenter.requestAuthorization` が ad-hoc 署名の
アプリでは常に拒否される環境がある。さらに `.app` バンドル外から
バイナリを直接実行すると初期化で例外を投げる。

**対策**: `Bundle.main.bundleIdentifier != nil` を確認してから初期化し、
許可が取れない場合は `osascript -e 'display notification ...'` に
自動フォールバックする 2 段構えにした。根本解決は Developer ID 署名 + 公証。

## 5. `defaults write ClawnPet ...` が効かない

**症状**: テスト用に `defaults write ClawnPet clawn.collapsed -bool false` と
書いても、バイナリ直接実行のインスタンスに反映されない。

**原因**: `.app` バンドル内のバイナリを直接実行しても `Bundle.main` は
バンドルを解決するため、UserDefaults のドメインはプロセス名ではなく
**`com.sunagaku.clawnpet`**（CFBundleIdentifier）になる。

**対策**: defaults の読み書きは常に `com.sunagaku.clawnpet` に対して行う。
逆に言うと build/ のテスト起動と /Applications の常駐版は
**同じ設定・同じ位置アンカーを共有する**ので、テストで動かした位置が
本番に残ることがある（実際に残った）。テスト後は値を戻すこと。

## 6. zsh の単語分割でアイコンが全部 1024px になる

**症状**: `for spec in "icon_16x16.png 16" ...; do set -- $spec; ...` という
bash 流のループが zsh では分割されず、`$1` に「名前 サイズ」全体が入って
全ファイルが同名+デフォルトサイズで生成された。`iconutil` は
「Failed to generate ICNS」としか言わないので原因に気づきにくい。

**対策**: zsh は `SH_WORD_SPLIT` が無効（未クォート変数を分割しない）。
`${=spec}` を使うか、ループをやめて 1 行 1 コマンドで明示的に書く。
iconset のファイル名は `icon_<size>x<size>[@2x].png` 規約から
1 つでも外れると全体が失敗する。

## 7. 常駐ペットは Computer Use から見えない・操作できない

**症状**: Claude の Computer Use（画面操作）でペットの UI を検証しようとしたが、
(a) セッション中に作ったアプリは request_access のアプリ一覧
（サーバー接続時のスナップショット）に載らず許可できない、
(b) 許可外アプリはスクリーンショットからコンポジタレベルで除外される、
(c) クリックも対象ウィンドウの owner が許可外だと拒否される。

**対策**: UI 検証は 3 点セットで代替した。
- **自己スナップショット**（SIGUSR1 → PetView が自分を PNG に描く）で見た目
- **`CGWindowListCopyWindowInfo`** で位置・サイズの実測（owner 名は
  CFBundleDisplayName の「Clawnくん」であることに注意）
- **CGEvent の合成クリック**（Bash から swift ワンライナー）で実操作

ただし合成クリックは**ユーザーが実際にマウスを動かしていると競合**し、
down→up の間にカーソルが流れてドラッグ判定になる（実際にペットを
1200px 引きずってしまった）。ユーザーが操作中のマシンでは使わないこと。

## 8. performDrag 委譲中はマウスイベントが来ない

**症状**: ドラッグ中の移動速度（「進行方向を向く」演出用）を
`mouseDragged` で取ろうとしたが、自前ドラッグを
`window.performDrag(with:)` に委譲した後はイベントが一切届かない。

**対策**: 30fps の描画 tick で**ウィンドウ位置のフレーム間差分**から速度を推定する。
副次効果として、プログラム的な移動にも自然に反応する。

## 9. transcript (JSONL) パースの罠

- `content` に `tool_result` を含む **user 行はエージェントの継続実行**であって
  人間の発話ではない。人間の発話は content が文字列 or `{type:"text"}` 配列のもの。
- `isMeta: true` の行と `<command-name>` など `<` 始まりのテキストはシステム由来。
  実況に出すと事故る（コマンド実行のたびに「かんがえ中」になる等）。
- 新規追跡時に末尾 16KB をバックフィルするが、**timestamp が 20 秒より古い行は捨てる**。
  これをしないと起動のたびに過去ログで celebrating が発火する。
- `cwd` はどの行にも入りうる。カードのタイトルは cwd の末尾ディレクトリ名。

## 10. Claude Desktop のセッションジャンプはパラメータ名が `session`

`claude://resume?sessionId=...` でも `claude://resume/<uuid>` でもなく、
**`claude://resume?session=<uuid>`**。app.asar 内のハンドラ
（`Resume deep link: missing or invalid session` のログ文字列周辺）から特定した。
成功すると Desktop は CLI transcript を取り込んで
`Imported CLI session … → setFocusedSession` まで自動でやってくれる。
CLI / Cursor 拡張のセッションでも Desktop 上で開ける。

## 11. 起動直後はバッジ（セッション数）が出ない

**症状**: 起動→即クリック検証をすると、バッジ判定が空振りして
「なでる」になる。

**原因**: 追跡対象の判定は「transcript への**起動後の追記**」を 1 秒ポーリングで
検知して初めて track が生える。起動直後の数秒は `tracks` が空で、
バッジも表示されずヒットテストも無効。

**対策**: 検証は起動から 3〜4 秒待ち、対象セッションに 1 行追記してから行う。

## 12. Finder のアイコンキャッシュ

`.app` を差し替えてもアイコンが古いまま表示されることがある。
`touch /Applications/ClawnPet.app` で LaunchServices の更新を促すと反映が早い
（それでも粘る場合があるが、実害はないので放置でよい）。
